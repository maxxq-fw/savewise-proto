# SaveWise Protocol — децентрализованный протокол микронакоплений с AI-прогнозированием бюджета

**SaveWise Protocol** — это гибридная Web3-платформа для управления личными накопительными целями. Проект объединяет frontend-интерфейс, backend API, blockchain-логику, utility-токен SAVE, marketplace и AI-модуль прогнозирования бюджета.

Сервис позволяет пользователю создавать накопительные цели в ETH, пополнять их, получать SAVE за выполнение плановых этапов накопления, использовать токены внутри marketplace и получать AI-прогноз вероятности достижения цели.

## Идея проекта

Основная идея SaveWise Protocol — создать сервис, который помогает пользователю регулярно откладывать средства на конкретные цели.

Пользователь может создать накопительную цель, указать сумму, срок и категорию, после чего постепенно пополнять ее. SAVE-токены начисляются не за каждую произвольную транзакцию, а за достижение плановых этапов накопления, рассчитанных по сроку и целевой сумме. Эти токены можно использовать внутри сервиса: покупать бонусы, активировать Reward Boost, использовать No-Penalty Pass и получать marketplace-купоны.

Проект решает проблему нерегулярных накоплений. Часто пользователь начинает копить, но теряет мотивацию, не видит последствий нарушения плана и не понимает, насколько реалистично достичь цели к выбранной дате. SaveWise объединяет накопительные цели, токеновую мотивацию и AI-прогнозирование, чтобы сделать процесс более понятным и управляемым.

## Актуальность

Актуальность проекта связана с несколькими факторами:

- пользователям нужен понятный инструмент для контроля личных накоплений;
- обычные трекеры бюджета не создают сильной мотивации к регулярным действиям;
- blockchain позволяет сделать правила начисления наград и списания токенов прозрачными;
- AI-аналитика помогает оценить вероятность достижения цели;
- marketplace превращает токен из абстрактной награды в инструмент получения конкретных преимуществ.

SaveWise показывает, как blockchain и AI могут применяться в повседневном финансовом сценарии, а не только в сложных DeFi-продуктах.

## Пользовательский сценарий

Базовый сценарий работы пользователя выглядит следующим образом:

1. Пользователь регистрируется в системе.
2. Создает накопительную цель.
3. Указывает целевую сумму в ETH и дедлайн.
4. Пополняет цель.
5. Получает SAVE за депозит.
6. Следит за прогрессом накопления.
7. Формирует AI-прогноз по цели.
8. Использует SAVE в marketplace.
9. При необходимости покупает No-Penalty Pass или Reward Boost.
10. Завершает цель или выполняет досрочный вывод.

## Токен SAVE

**SAVE** — utility-токен проекта. Он используется как внутренняя единица мотивации и доступа к функциям сервиса.

Токен не позиционируется как инвестиционный актив. Его задача — связывать финансовую дисциплину пользователя с практическими возможностями внутри продукта.

### Как пользователь получает SAVE

Пользователь получает SAVE за полезные действия:

- стандартный плановый этап накопления начисляет **+5 SAVE**;
- плановый этап при активном Reward Boost начисляет **+10 SAVE**;
- завершение цели может давать дополнительный бонус.

### Как пользователь тратит SAVE

SAVE можно тратить на:

- **Reward Boost**;
- **No-Penalty Pass**;
- marketplace-купоны;
- штраф при досрочном выводе без пропуска.

### Экономический смысл SAVE

SAVE создает внутренний цикл мотивации:

- пользователь копит;
- получает SAVE;
- использует SAVE для преимуществ;
- возвращается к целям и продолжает пополнять их.

Таким образом токен связывает действия пользователя с системой поощрений и расходных сценариев.

### Защита от бесконечной эмиссии SAVE

В проекте используется ограничение на начисление SAVE. Награда не выдается за каждое пополнение автоматически. Для каждой цели рассчитывается плановый этап накопления:

```text
target amount / количество периодов до дедлайна
```

Например, если пользователь хочет накопить **100 ETH за 10 недель**, то один плановый этап равен **10 ETH**. Пользователь может пополнять цель по 1 ETH, но SAVE будет начислен только после достижения очередного порога в 10 ETH. Это предотвращает ситуацию, когда пользователь дробит одну цель на множество мелких депозитов и бесконечно получает SAVE.

Если расчетный этап получается меньше 1 ETH, применяется минимальный этап 1 ETH, потому что минимальное пополнение в системе также составляет 1 ETH.

## Reward Boost

Reward Boost — это marketplace-привилегия, которая усиливает награды за будущие депозиты.

Пользователь покупает Reward Boost за SAVE. После покупки следующие **5 плановых reward-этапов** дают повышенную награду:

- обычный reward-этап: **+5 SAVE**;
- reward-этап с Reward Boost: **+10 SAVE**.

Такая механика делает Reward Boost экономически осмысленным. Пользователь получает выгоду не сразу, а только если продолжает регулярно пополнять накопительные цели. Это стимулирует долгосрочное поведение и повышает регулярность использования сервиса.

---

## No-Penalty Pass

No-Penalty Pass — это привилегия, которая позволяет один раз выполнить досрочный вывод без стандартного штрафа.

В обычном сценарии досрочный вывод требует списания 10 SAVE. Если у пользователя есть No-Penalty Pass, он может закрыть цель досрочно без этого штрафа.

Эта механика делает SAVE полезным не только для бонусов, но и для управления рисками пользователя. Пользователь может заранее приобрести защиту от штрафа, если хочет сохранить гибкость.

---

## Marketplace

Marketplace — это раздел, где пользователь может тратить SAVE на внутренние бонусы и купоны.

В marketplace представлены:

- Reward Boost;
- No-Penalty Pass;
- скидка 10% в Пятёрочке на покупку от 1000 ₽;
- сертификат на кофе;
- +1 категория кешбэка в Т-Банке.

После покупки пользователь получает персональный код купона. Marketplace нужен для того, чтобы у SAVE были понятные способы расходования внутри продукта.

## AI-модуль

AI-модуль используется для прогнозирования вероятности достижения накопительной цели.

Пользователь вводит бюджетные параметры:

- ежемесячный доход;
- обязательные расходы;
- переменные расходы.

Backend также использует данные цели:

- target ETH;
- current ETH;
- deadline;
- статус цели;
- прогресс накопления.

На основе этих данных AI-модуль формирует прогноз.

### Результат AI-прогноза

AI-прогноз возвращает:

- вероятность достижения цели в процентах;
- уровень риска;
- рекомендуемый ежемесячный депозит;
- рекомендуемый еженедельный депозит;
- прогнозируемую дату достижения;
- краткое резюме;
- практические рекомендации.

### Техническая реализация AI

AI-модуль реализован на backend через **Groq SDK**.

Используется Llama-модель:

- **llama-3.1-8b-instant**

Backend формирует prompt, отправляет данные цели и бюджета во внешний AI API и получает структурированный JSON-ответ. Это позволяет использовать AI как отдельный аналитический модуль, не смешивая его с frontend-логикой.

## Смарт-контракты

В проекте используются два основных смарт-контракта:

- `SaveToken.sol`;
- `SavingsVault.sol`.

### SaveToken.sol

`SaveToken` реализует ERC-20 токен SAVE.

Основные задачи контракта:

- выпуск SAVE;
- хранение балансов пользователей;
- ограничение mint-операций через роли;
- сжигание SAVE при штрафах и покупках;
- предоставление токеновой основы для marketplace и rewards.

Контракт использует OpenZeppelin ERC-20 и AccessControl.

### SavingsVault.sol

`SavingsVault` реализует основную бизнес-логику накоплений.

Основные задачи:

- создание цели;
- хранение параметров цели;
- прием депозитов;
- начисление SAVE;
- завершение цели;
- досрочный вывод;
- применение штрафов;
- использование No-Penalty Pass;
- применение Reward Boost;
- покупка marketplace-привилегий.

- `GoalCreated`;
